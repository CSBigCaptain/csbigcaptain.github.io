转眼间 2025 年也要到最后一天了。2025 过得很快，快到我常常来不及记录，就已经被推着往前走。回头看的时候才发现，这一年里我经历了很多细碎的变化。我想把这一年写下来，不是为了总结得多漂亮，只是想认真地对自己说一句：辛苦了。

## 回顾与复盘

### 技术提升

这一年在技术提升上，我投入了很多精力，也收获了不少回报。我接触并体验了许多技术栈，认识了许多志同道合的朋友，这里面也不乏一些翘楚，他们给予了我莫大的帮助。遇到好的朋友能够改变自己的命运，这句话在我身上终于得到了实践。不过我觉得今年只是一个开始，~~明年我肯定会更加引以为傲~~。

这一年，我接触到了许多新的技术栈，其中 Nuxt UI 是我今年最喜欢的 UI 库之一。随着 Nuxt UI 与 Nuxt UI Pro 合并并全面开源，我也得以免费使用完整的组件与模板体系。我喜欢它，不仅因为组件覆盖的场景足够广，更因为它提供了几套开箱即用的模板：从首页、管理后台，到 AI 聊天页、博客等，都能直接拿来改造。对我来说，这些模板带来的价值甚至超过了组件本身——我在“照着做”的过程中学到了不少 Nuxt 的生产实践，也更清楚一个项目从页面到结构应该怎样组织。

### AI 工具：一场“工业革命”

年初我接触到了字节的 Trae 国际版，感觉非常惊艳：原来 AI IDE 能把写代码这件事变得这么顺。它不仅能补全代码、定位报错，还能自动检查并给出修改建议——那段时间我几乎离不开它。后来同学提醒我 VSCode 的 Copilot 也能做到类似效果，加上我有 GitHub Education 赠送的会员，于是又回到了 VSCode。

到了下半年，我基本每个月都会订 GPT Plus，试了他们家的 VSCode 插件，再次被惊艳到：Codex 甚至能读到 `node_modules` 里的代码，给出的建议因此更精确、可靠。得益于这些工具，我下半年博客/项目的推进明显快了很多。

当然，“碳基生物”和“硅基生物”之间可能天然地存在一些“代沟”，一开始我与 AI 的交流并不算顺畅。后来我逐渐意识到，问题往往不在于 AI，而是我们自己——我们表达的需求中默认了太多的背景信息，而这些信息 AI 是不知道的，也不会被他们“自动补全”。简而言之就是：想让 AI 给出更可靠的结果，关键是把需求说清楚——越具体越好。

总的来说，这场“工业革命”更像是《赛博朋克 2077》中的外骨骼：它不是替我走路，而是把我的各项属性“放大”。它也像是我的后勤助理：帮我处理琐事，专注于工作与创意。

### 新朋友

今年我最大的收获之一，是在社区里认识了一些非常优秀、也非常愿意分享的朋友。十月份我在 MDUI 交流群中咨询问题时认识了纸鹿，后来又在他的交流群结识了御守（Vue 团队成员，也是 Vue Language Tools 的主要贡献者）等人。他们在我遇到各种具体问题时总能给出耐心、可靠的建议，有些甚至是“顺手一提”就帮我省下了大量试错时间。

就在前几天，我又结识了远方等人，他们也给予了我莫大的支持。

回头看，这些善意不仅帮我解决了问题，也让我更确信：在技术这条路上，和对的人同行，很多事情会容易很多。

### 新玩具

今年下半年开学的时候用外包赚到的钱买了一台 Insta 360 Ace Pro。起初购买它一方面想拍着玩，另一方面是买数码产品的时候能够记录开箱过程出现问题有利于维权。刚到手的时候出去玩的时候就会挂在脖子上打开录制记录生活，后来热情就消减了。再加上这类运动相机的对焦距离很远，没办法拍清楚物品具体的细节，假如物品有磕碰划痕什么的，运动相机很难拍出来。因此目前基本吃灰了…

今年暑假买了一条雷蛇的战锤狂鲨 V3 耳机，听感感觉与我之前的水月雨夸克 2 区别不是非常大。关于这个耳机有两个趣事儿值得分享：其一是我之前用口香糖清理耳机的防尘网，结果口香糖粘在防尘网上了，幸运的是听感没有什么变化。另外一件是，我以为这个耳机的防尘网和耳机腔体是一体的，直到后面我不小心把防尘网撕下来才发现防尘网是一个铁片用胶粘在腔体上的。虽然我趁着胶水黏性还在又粘回去了，但是我担心黏性肯定会大打折扣，有脱落的可能，目前买不到这款耳机的防尘网，如果脱落丢失的话可能就非常麻烦了。

### 测试与竞赛

今年是我在大学中第一年参加一些竞赛和测试，但整体都属于“参与了、却没有真正投入”的状态。简单来说，我更像是在走流程，这与比赛“提升自己”的初衷并不一致。

今年四月份，我第一次参加蓝桥杯省赛，由于当时自己比较松懈，并没有系统性的备赛，因此只取得了省二，成绩自然是远不及身边的朋友。

六月份是英语四级考试，这是我第一次参加四级考试，同样由于没有专心准备，我近乎是裸考，只是每天用多邻国简单维持语感。幸运的是我考了 450 分左右。尽管成绩可能不是很理想，但是对比考了三次都没过的同学，我算是比较幸运的。

后来在辅导员的推荐下，我又参与了睿抗比赛，最终拿到了国赛三等奖。只是大部分工作由研究生队长完成，我并没有做什么工作；而国赛线下赛点在杭州，加上时间和经费的限制，我也没能到现场参与。回头看，我在团队里的角色更多是学习和配合，这次经历也让我更直观地看到差距：如果想在竞赛上真正有所收获，投入和准备依然是绕不开的。

## 生活上的回顾

### 音乐

得益于 Apple Music 的年度总结功能，我今年也能看到属于自己的年度回忆。最让我意外的是，听得最久的歌手还是邓紫棋。暑假时我开始听塞壬唱片的作品，算是今年在音乐上的一个新发现。不得不承认，鹰角网络在音乐这一块的水准确实很能打——无论是编曲、音色还是整体制作质感都很稳，也难怪大家会把它戏称为“音角”。尤其是 12 月那首和 OneRepublic 合作的《明日方舟：终末地》印象曲，是真的顶——我后来几乎是忍不住一直单曲循环。

### 游戏

我今年体验了不少新游戏，这里挑几款简单聊聊。

#### 燕云十六声

年初刚接触《燕云十六声》时，我的第一反应是“惊艳”。它把古代武术兵器的风格与近几年流行的开放世界探索结合在一起，整体气质很新，玩起来也确实有新鲜感。

更重要的是，它在设计上对玩家相当友好。比如几乎不需要“爆肝”：日常做点任务就能兑换成长与突破资源；清理几遍 BOSS 还能换到金色品质武器自选包。体力系统也比较宽松——上限 480，0 到满需要 48 小时，实际游玩节奏很容易保持在“想玩就玩，不想玩也不焦虑”的状态。游戏里各种武学也主要靠任务或门派获取，而不是把核心内容绑定在抽卡上。

内容层面也有不少有趣的设计：大地图里的钓鱼、投壶等玩法都比较丰富；大世界既能单人探索，也能多人联机互动，打不过的 BOSS 可以邀请老玩家来帮忙。联机模式有点儿像逆水寒，大广场互动加聊天。

值得注意的是，作为一款开放世界游戏，它的提升强度并不依赖抽卡。商业化更多集中在稀有皮肤和武器样式锻造上，整体体验可以说是“对普通玩家很友好”。

虽然我在五月份就退坑了，年底也经历了一些运营争议，但如果是刚入坑、想找一款“开放世界 + 武侠”且不太逼氪逼肝的游戏，我依然会推荐它作为体验选择。

#### 明日方舟

《明日方舟》是我今年接触到最喜欢的游戏。它最打动我的地方在于：核心玩法成熟、策略性足够强，而且数值膨胀控制得相对克制——即使玩了很久，也不太会出现“新内容完全把旧体系淘汰掉”的失衡感。

游戏会不定期出一些有趣的限时玩法：双人联机的促融共竞，自走棋模式卫戍协议、“赛博斗蛐蛐”争锋行动等等。

当然，争议也一直存在。它的玩家群体和其他二游一样分裂比较明显，游戏运营节奏不断。虽然干员基础出率较高，但是保底门槛并不低，会让一部分经常歪的玩家在抽卡体验上比较受挫。虽然现在游戏主线关卡推进简单，但部分高难度活动关卡也会更依赖卡池深度，新人或回坑玩家如果角色储备不足，很容易有挫败感。

另外它的美术和音乐整体都很强，风格统一且完成度高。尤其是 UI 与平面设计这一块，从信息层级到字体与版式细节都很讲究，很多地方甚至能当作设计参考来拆解。鹰角网络旗下的塞壬唱片出品的歌曲也收获了很多玩家的喜爱。

总体而言，我依然认为《明日方舟》是一款值得投入的长期游戏：它适合喜欢策略、愿意慢慢积累、并且能接受抽卡不确定性的玩家；如果你更偏向轻量、追求即时爽感，或者很在意卡池体验，那它可能会显得门槛偏高。

#### 少女前线 2：追放

在我心目中，《少女前线 2：追放》是一款偏轻量、节奏很休闲的“收菜型”游戏。虽然上线初期问题不少，但整体福利不算少，抽卡节奏也相对友好——至少相比一些同类二游，它不会让人有太强的“被催着上线”的压力。制作层面也挺能打：角色建模细节丰富，剧情延续了前作基调，因此依然能吸引一部分老玩家继续玩下去。

此外，它的“低负担”不仅体现在日常任务量上，也体现在玩法机制里：多数战斗可以交给系统接管，官方也提供了自动化刷材料的功能。对我来说，它更像是一款适合碎片时间打开、顺手清日常的游戏——如果你追求轻量体验，或者更在意剧情和角色，那么它会是一个比较舒服的选择。

### 一些小挫折

今年 11 月底，我向鹰角网络投递了两份实习简历，结果都被拒了。我没有太意外：地点在上海，加上当时学业也比较紧，就算通过可能也很难真正去应聘。只是这件事还是让我更清楚地看到差距——简历上能拿得出手的东西还不够多，后面得更认真地补上。

## 2026 年的计划

今年我学到了不少前端工程化的知识，2026 年我想把这些知识真正落到项目里，同时补齐后端与数据库这块的短板。

- Carbon Chat：这是一个 AI 聊天网站，目前通过 DeepSeek、火山、智谱等平台的 API 实现聊天能力，现阶段主要在测试和打磨功能。2026 年我希望把它继续完善，逐步补上云端消息、账号体系和管理员后台等模块。
- mdui-vue：一个基于 Reka UI 设计的 Material Design 组件库，计划同时支持 Vue 和 Nuxt。我的博客目前使用的是 mdui，但在生产实践中也遇到了一些明显的痛点：一方面服务端渲染不够友好，页面加载时容易出现样式跳动；另外 Web Components 的 Shadow DOM 在样式覆盖上也比较难控。当然，这个项目由于对于我来说太难，也可能会胎死腹中。
- 值得注意的是，今年对于我来说是考研年。寒假结束后课程依然很多，这对学习节奏会是一个不小的阻力。所以我也不再给自己设定过高的预期，能稳住进度、把该做的事情持续做下去，就已经足够了。

回头看，2025 并不是发生了什么大事的一年，但它让我在很多细节上悄悄变得更成熟：更会使用工具，也更愿意向社区求助；更能接受自己的不足，也更清楚自己接下来要补什么。2026 我不打算给自己许下太宏大的承诺，只希望把节奏稳住，把计划里的事情一点点落到地上。至于结果，就交给时间。
